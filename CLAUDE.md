## What This Is

Stellaris is a Hugo theme for technical writing and blogging. It's distributed as a Hugo module (no compilation step). The example site lives in the repo root alongside the theme files.

## Development Commands

```bash
# Start local dev server
hugo server

# Production build
hugo --minify

# Update Hugo module dependencies
hugo mod get -u

# Generate QR codes locally (requires Python venv)
python3 -m venv .venv && .venv/bin/pip install "qrcode[svg]" && .venv/bin/python3 .github/scripts/generate_qr.py "https://your-base-url"
```

Requirements: Hugo >= 0.146.0 (standard build, extended NOT required), Go (for module management).

## Architecture

Hugo theme structure with these key areas:

**Layouts** (`/layouts/`): Templates drive all rendering.
- `baseof.html` — top-level shell (header + sidebar + main + footer)
- `home.html` — homepage: hero + two-card row (featured topic / latest post) + recent posts list + topics grid
- `page.html` — individual post/article with optional series strip and QR code
- `section.html` — section index showing subsection cards + post lists
- `_partials/` — reusable fragments (head, header, sidebar, toc, math, project-posts, qr)
- `shortcodes/code.html` — syntax-highlighted code block with optional filename label

**Assets** (`/assets/`): Processed by Hugo Pipes.
- `css/main.css` — all styles; neobrutalist dark theme using CSS custom properties
- `js/main.js` — TOC scroll highlighting via IntersectionObserver

**Content model**: Two content types:
- `/content/posts/` — standalone posts (flat)
- `/content/topics/<slug>/` — topic series; `_index.md` defines the topic, numbered files are the posts. A "series strip" appears on posts when a topic has 2+ entries.

**QR codes** (`/static/qr/`): SVG QR codes generated by `.github/scripts/generate_qr.py`, gitignored and regenerated on every CI build. Grey (`#888888`) fill, transparent background.

## Key Configuration (`hugo.toml`)

Notable `[params]`:
- `accentColor` — CSS accent color (hex); also drives inline code block colours automatically via `color-mix()`
- `featuredTopic` — slug of the topic pinned in the homepage featured card
- `fontSans` / `fontMono` — font family names (must be available in `/static/fonts/` or system)
- `heroTitle` — typewriter text on the homepage hero

Syntax highlighting uses Chroma with `noClasses = false` — highlight classes map to CSS tokens defined in `main.css`. Math rendering uses self-hosted KaTeX (opt-in per page with `math = true` in front matter).

## Styling Conventions

- **Neobrutalist dark theme**: dark bg (`#1a1a1a`), light borders (`#e0e0e0`), hard offset box-shadows (`4px 4px 0 var(--border)`), zero border-radius, heavy typography (700–900 weight)
- Accent color is set in `hugo.toml` and injected as `--accent` via an inline `<style>` block in `head.html` — overrides the fallback in `main.css`
- Background noise overlay at 15% opacity via `body::after` SVG filter
- Responsive breakpoints: ≤1199px drops TOC, ≤767px collapses sidebar to overlay drawer (CSS-only via `:has()`)
- No external CDNs — fonts and KaTeX are fully self-hosted in `/static/`
